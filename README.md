# 📦 Simple WO — Mini Work Order System

A mini **Work Order System**, built as a study project with a focus on solid programming practices, software architecture, and design patterns.

---

## 📖 About the Project

This project was created to **study and apply modern software development practices** through the construction of a real Work Order (Service Order) system, inspired by real systems used in the market.

The core idea isn't to build a ready-to-use commercial product, but to use a real, well-known domain — work order management — as a practical foundation for deepening technical knowledge: architecture, code organization, design patterns, and framework best practices.

The project is developed with **PHP + Laravel**, simulating a real production environment, while intentionally keeping the scope lean and focused on the essentials.

> 🎓 **This is a study project.** There is no intention of releasing it as a final product or commercial solution. The focus is on the learning process, not on delivering a finished product.

## 🧩 Functional Scope

Since the focus is on learning rather than delivering a complete product, the system covers only the **essential features** of a Work Order System, such as:

- Creating and managing work orders
- Tracking status (open, in progress, completed, etc.)
- Basic customer and service registration
- User authentication

More complex or commercial-specific features (advanced billing, multiple integrated modules, advanced reporting, etc.) are outside the scope of this project.

## ⚙️ Installation

> **Requires:** Docker

### 1. Clone the repository
```bash
git clone https://github.com/LeandroMF20/mini-erp-WO.git
cd Simple-OS
```

### 2. Run the installer
```bash
chmod +x install.sh
./install.sh
```

Once finished, the application will be available at **http://localhost:8080** and the MySQL database at **localhost:3306**.
